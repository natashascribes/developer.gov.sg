Overview

Modern web applications commonly use REST APIs to faciliate communications between programs. The following API playbook aims to encourage consistency, maintainability and good API practices within the Singapore Government so that development teams can easily leverage on conventions and standards to build digital services more quickly and intuitively. The API playbook has so far been maintained by the Apex team.

These guidelines represent one suggested way of designing REST web-based APIs for most scenarios in the Government. Depending on your use case, there could be other principles more suited for you that might not be covered in this guide.
